# Envision 2026 — MD Simulator Tasks

A task-based repository for learning GitHub workflows, with LAMMPS simulation files coming later in the program.

---

## Submission Instructions

1. Fork this repository — [fork here](https://github.com/Skanda09118/Envision_2026_MD_Simulator_Tasks/fork)
2. Create a branch with your name
3. Complete the tasks below
4. Push and open a Pull Request

---

## Tasks

### Task 1 — Fork and Clone

Fork this repository to your own GitHub account, then clone it to your local machine.

Figure out how to do this using GitHub's documentation or a quick search.

---

### Task 2 — Create a Branch

Before making any changes, create a new branch named after yourself and switch to it.

Do not work directly on main.

---

### Task 3 — Add Your Entry

Create a file named `participants_YourName.md` and add your answers to these:

- One food you could eat every day without getting bored
- Something everyone loves that you just don't get
- The most useless superpower you would still choose

Keep it honest.

---

### Task 4 — Commit and Push

Stage your changes, write a clear commit message, and push your branch to your fork.

A good commit message describes what you changed, not how.

---

### Task 5 — Open a Pull Request

Once pushed, open a Pull Request from your fork back to this repository. Write a short description of what you did in the readme file .

---

### Task 6 — Review Someone's PR

Find an open Pull Request from someone else and leave a meaningful comment or suggestion. Approving without reading doesn't count.

---

### Task 7 — Open an Issue

Use the Issues tab to flag something unclear, suggest a new task, or propose how LAMMPS files should be organised in this repo.

---
### Task 8 — Learning basic Linux commands

> Learn how to change directories , create new files and other basic linux navigation tasks.
